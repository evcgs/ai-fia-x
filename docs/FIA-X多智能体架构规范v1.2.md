# FIA-X 多智能体创新架构规范 v1.2

基于FIA-X三大支柱：**First-principle（第一性原理）+ Interdisciplinary（跨学科交叉）+ Anti-consensus（反共识思维）**

**遵循OIA顶层协同框架：** OIA (One Intent, All Agents) 多智能体协同执行标准

---
## 🏗️ 整体架构设计

### 分层架构（v1.2优化）

```
┌─────────────────────────────────────────────────────────────┐
│ 🔝 顶层：OIA六步闭环协同框架 (必须遵循)                            │
│  用户请求 → 意图解析 → 全局感知 → 任务分发 → 并行执行 → 交叉验证 → 结果汇总 → 质量审核 → 交付用户 │
├─────────────────────────────────────────────────────────────┤
│ 🎯 中层：FIA-X专业创新执行层 (专业分工)                            │
│  七个专业智能体，每个专注一个创新环节，专业化分工最大化                │
├─────────────────────────────────────────────────────────────┤
│ 📦 底层：基础设施 (MemOS记忆 + 飞书消息 + 网关)                      │
│  每个智能体独立飞书App ID + 独立MemOS记忆，隔离互通                  │
└─────────────────────────────────────────────────────────────┘
```

**设计原则：**
- 顶层OIA管协同：遵循OIA强制规则，保证协同质量
- 中层FIA管创新：七个专业智能体专注创新环节，发挥专业能力
- 底层基础设施：保证记忆隔离和消息通信

## 🔗 完整协同流程（优化后v1.2）

### OIA顶层流程 + FIA专业执行流程

```
用户请求  →  OIA顶层启动
 ↓
1️⃣ OIA：意图解析 → 分配ProblemAgent
 ↓
2️⃣ FIA：ProblemAgent → 问题本质拆解 + 约束识别 + 问题重定义
    输出：《问题定义报告》
    ✅ 检查点：问题是否拆解到底层，约束是否识别完整
 ↓
3️⃣ OIA：全局感知 → 分配KnowledgeAgent
 ↓
4️⃣ FIA：KnowledgeAgent → 跨领域知识扫描 + 方法迁移适配
    输出：《跨领域知识迁移报告》
    ✅ 检查点：是否找到可迁移的成熟方法，知识缺口是否识别
 ↓
5️⃣ OIA：任务分发 → 分配HypothesisAgent
 ↓
6️⃣ FIA：HypothesisAgent → 主流共识识别 + 逆向假设生成 + 高价值筛选
    输出：《反共识创新假设报告》
    ✅ 检查点：是否真正反共识，假设是否可验证
 ↓
7️⃣ OIA：任务分发 → 分配SolutionAgent
 ↓
8️⃣ FIA：SolutionAgent → 多方案生成 + 模拟推演 + 迭代优化
    输出：《落地方案报告》（多方案+推荐方案）
    ✅ 检查点：是否生成多个差异化方案，推荐方案是否有充分论证
 ↓
➕ 9️⃣ FIA：**RiskAgent → 风险预审** 【新增环节】
    输出：《风险预审报告》
    ✅ 检查点：识别高风险方案，预审不通过直接淘汰
    🎯 价值：提前淘汰高风险方案，避免浪费后续环节成本
 ↓
1️⃣0️⃣ OIA：任务分发 → 分配EvaluationAgent
 ↓
1️⃣1️⃣ FIA：EvaluationAgent → FIA量化评分 + 风险量化 + 方案排序
    输出：《量化评估报告》
    ✅ 检查点：评分维度完整，风险量化清晰
 ↓
➕ 1️⃣2️⃣ FIA：**ProblemAgent → 问题一致性复核** 【新增环节】
    输出：《一致性复核报告》
    ✅ 检查点：验证推荐方案是否真的解决了原始问题
    🎯 价值：避免"方案漂亮，但解决假问题"，确保聚焦真问题
 ↓
1️⃣3️⃣ OIA：任务分发 → 分配ExecutionAgent（FIA内部协调）+ PMAgent（项目管理）
 ↓
1️⃣4️⃣ FIA：ExecutionAgent → FIA内部任务分解 + FIA智能体分配
    输出：FIA内部执行计划
 ↓
1️⃣5️⃣ OIA：PMAgent → 整体项目计划 + 里程碑设定 + 外部任务分配
    输出：《项目执行计划表》
 ↓
1️⃣6️⃣ FIA：**LearningAgent → 阶段小结 + 最终沉淀** 【优化环节】
    - 每个阶段完成后做一次小总结
    - 项目完成后做整体案例沉淀和体系优化
    输出：《案例沉淀报告》+《体系优化建议》
 ↓
1️⃣7️⃣ OIA：交叉验证 → TestAgent质量验证 + RiskAgent合规检查
 ↓
1️⃣8️⃣ OIA：结果汇总 → PMAgent汇总结果
 ↓
1️⃣9️⃣ OIA：质量审核 → PMAgent最终审核
 ↓
🔚 OIA：交付用户
```

## 📋 各智能体详细职责边界

### 1. 🔍 ProblemAgent - 问题定义专家
**核心定位**：因果建模专家，剥离经验假设，找到问题本质

**核心职责**：
- 按照第一性原理拆解问题到最小单元
- 梳理因果链路，识别核心矛盾
- 识别所有显性/隐性约束条件
- 重新表述问题，找到真问题
- 将模糊目标量化为可验证目标
- **新增职责**：方案完成后做一致性复核，确保方案解决真问题

**边界限制**：
- 不负责知识检索 → 交给KnowledgeAgent
- 不负责方案生成 → 交给SolutionAgent

---
### 2. 📚 KnowledgeAgent - 跨领域知识专家
**核心定位**：跨领域知识迁移工程师，打破领域壁垒

**核心职责**：
- 跳出当前领域，扫描相关领域成熟方法
- 将其他领域方法迁移适配到当前问题
- 建立知识图谱索引，快速定位相关知识
- 多领域知识融合，产生组合创新
- 识别知识缺口，明确缺失信息

**边界限制**：
- 不负责问题本质拆解 → 交给ProblemAgent
- 不负责假设生成 → 交给HypothesisAgent

---
### 3. 💡 HypothesisAgent - 反共识创新专家
**核心定位**：共识挑战者，破局思维激发者

**核心职责**：
- 梳理行业主流共识和常规做法
- 站在共识反面思考，生成反共识创新假设
- 评估反共识机会价值，筛选高价值机会
- 将粗略假设细化为可验证创新框架
- 打破路径依赖，找到差异化创新路径

**边界限制**：
- 不负责知识收集 → 交给KnowledgeAgent
- 不负责方案细化 → 交给SolutionAgent

---
### 4. 🛠️ SolutionAgent - 方案创新工程师
**核心定位**：创新方案建筑师，将假设落地为可执行方案

**核心职责**：
- 基于创新假设生成多个差异化候选方案
- 对每个方案进行沙盘模拟推演
- 多维度比较不同方案优缺点
- 对优选方案多轮迭代优化
- 设计具体落地实施路径和里程碑

**边界限制**：
- 不负责量化评估 → 交给EvaluationAgent
- 不负责整体项目管理 → 交给PMAgent

---
### 5. 🛡️ RiskAgent - 风险管控专家
**核心定位**：风险预审官，提前识别防控风险

**核心职责**（v1.1优化后）：
- **新增**：对SolutionAgent生成的方案进行风险预审
- 对最终方案进行全面风险评估
- 识别安全、合规、隐私风险
- 给出具体风险防控建议
- 检查方案合规性

**边界限制**：
- 不负责方案生成 → 交给SolutionAgent
- 不负责量化评分 → 交给EvaluationAgent

---
### 6. 📊 EvaluationAgent - 量化评估专家
**核心定位**：创新量化评分师，客观评估方案价值

**核心职责**：
- 按照FIA五维量化体系评分（创新性/可行性/价值性/可复制性/可进化性）
- 对方案进行风险量化评级
- 按照得分排序，推荐最优方案
- 针对低分维度给出具体改进建议
- 提供量化数据支持最终决策

**FIA量化评估体系**：

| 评估维度 | 权重 | 说明 |
|----------|------|------|
| **创新性** | 30% | 反共识强度、方案独特性、认知突破程度 |
| **可行性** | 25% | 资源适配、技术可实现、风险可控 |
| **价值性** | 20% | 商业价值、社会价值、用户价值 |
| **可复制性** | 15% | 标准化程度、可推广范围、复用价值 |
| **可进化性** | 10% | 学习空间、迭代潜力、持续优化 |

**边界限制**：
- 不负责方案生成 → 交给SolutionAgent
- 不负责落地执行 → 交给ExecutionAgent/PMAgent

---
### 7. 📋 ExecutionAgent - FIA执行协调专家
**核心定位**：FIA内部协调指挥官，协调FIA智能体执行

**核心职责**（v1.2优化后）：
- 对FIA内部任务进行分解
- 分配给对应FIA专业智能体
- 协调FIA智能体之间的协作
- 跟踪FIA内部任务进度
- 汇总FIA内部执行结果给PMAgent

**边界限制**：
- 整体项目管理对外交付 → 交给PMAgent（遵循OIA规则）
- 不负责专业领域具体工作 → 分给对应专业智能体

---
### 8. 📈 LearningAgent - 进化学习专家
**核心定位**：沉淀学习专家，驱动体系持续进化

**核心职责**（v1.1优化后）：
- **优化**：每个阶段完成后做阶段小结，及时记录经验
- 项目完成后将完整创新项目沉淀为案例
- 提炼成功方法论，总结经验教训
- 根据实践反馈优化FIA流程和方法
- 更新知识图谱和共识库，保持体系活力
- 驱动FIA体系持续进化

**边界限制**：
- 所有前置环节完成后才执行
- 负责沉淀总结，不负责前置创新环节

## 🚦 质量闸门规则（强制执行）

每个环节必须满足检查点要求，才能进入下一环节：

| 环节 | 质量闸门 | 不通过处理 |
|------|----------|------------|
| ProblemAgent | 问题必须拆解到底层，约束必须完整识别 | 返回重拆 |
| KnowledgeAgent | 必须找到至少2个跨领域可迁移方法 | 重新扫描 |
| HypothesisAgent | 必须生成至少一个真正反共识假设 | 重新思考 |
| SolutionAgent | 必须生成至少3个差异化方案 | 重新生成 |
| RiskAgent（预审） | 不能有不可控高风险方案 | 淘汰高风险方案，返回重新生成 |
| EvaluationAgent | 推荐方案必须得分≥70分 | 返回优化 |
| ProblemAgent（复核） | 方案必须一致解决原始问题 | 返回重设计 |
| PMAgent（审核） | 项目计划必须完整清晰 | 返回重制 |
| TestAgent（验证） | 配置必须正确，质量合格 | 返回整改 |
| RiskAgent（检查） | 必须合规，无高危风险 | 返回整改 |

## 🔐 核心设计原则

1. **OIA+FIA分层设计**：OIA管顶层协同，FIA管专业创新，各司其职
2. **专业化分工**：每个智能体只做一件事，专业能力最大化
3. **质量闸门左移**：尽可能早发现问题，降低返工成本
4. **一致性保证**：始终聚焦原始问题，避免偏离
5. **持续学习**：每个阶段都总结，持续优化
6. **完全遵循OIA**：严格遵守OIA任务分发规则，项目给PMAgent，文档给DocAgent
7. **记忆隔离互通**：每个智能体独立记忆存储，支持跨Agent检索
